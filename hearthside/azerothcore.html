AzerothCore

Wobblin' Goblin's AzerothCore for Docker Desktop

AzerothCore for Docker Desktop is a great way to start an AzerothCore server for yourself or for your family and friends. The only requirement is Docker DesktopDocker is a platform for helping people build, share and run applications. Docker Desktop is a graphical application for building and sharing applications and services on the Docker platform. which you will need to install to follow the tutorial.

Target Audience

We assume that you are familiar with playing on a private server and have a 3.3.5a client. We'll be relying on Docker Desktop and in-game commands to run our server so minimal technical expertise is required.

The Docker imagesA Docker image is a file used to execute code in a Docker container. A Docker container is a runnable instance of an image. provided by Wobblin' Goblin have opinionated defaults to allow for a good single player or small group experience. Think of our Docker images as a repack that is easily updated.

Prerequisites

You'll need Docker Desktop to get started. You can find installation instructions for WindowsOn Windows you'll want to use Windows Subsytem for Linux as Docker's backend. You can learn how to Install Windows Subsystem for Linux on Microsoft's website., Mac, and Linux on Docker's website. This is the hardest part of the tutorial so read the installation instructions carefully and if you run into any problems don't be afraid to ask the community for help.

Installation

Once you have Docker Desktop installing the server is done in three easy steps.

  1. Run Image

    In Docker Desktop go to the search bar in the top right (Ctrl+K) and search for wobgob/azerothcore. Once you've found the image you'll want to hit Run which will bring up a prompt. Do not hit Run again yet.

  2. Enter Settings

    In the prompt open Optional settings and make sure your settings match the below image then hit Run. Each setting is explained below.

    1. The container name can be anything. We've chosen to go with a randomly generated name.

    2. Host ports should match container ports

      Warning! Exposing port 3306 can be a security vulnerability as anyone with access to that port can access your database. Leave this blank if you're unsure.

    3. The host path can be anything. We've chosen to go with a volumeVolumes are the preferred mechanism for persisting data generated by and used by Docker containers. named after the container path.

      Warning! Do not delete your volume in Docker Desktop without backing it up or you'll lose all your data.

    4. The container path must be /var/lib/mysql

  3. Login

    You'll be taken to the server's logs where you'll need to wait for the server to initialize before you can login. First-time setup can take a few minutes.

    Set the realm list of your 3.3.5a client to 127.0.0.1 then use the username acore and password acore to log into the console account.

Setup

Once you're logged into the console account create a game master (GM) character with a name of your choice. This will be the character you'll manage the server with.

It's recommended that you use a separate player account for playing by yourself or with others. To create another account enter the following commandSee AzerothCore's website for a list of GM commands. into chat

.account create $account $password

Where you should replace $account with your account name and $password with your account's password.

You can now log into your player account and begin playing!

FAQ

How do I change the default configuration? You'll need to get a copy of the source code from our GitHub to create a new image. You'll want to download the hearthside-stories branch where you can find configuration files in etc and you can place new modules in modules. Finally run the following command from the root of your project

        docker build -t azerothcore .

To create a new image called azerothcore which you can then run.

How do I play with others? Unless you port forward or use a tunneling service you'll only be able to play with others in your local area network.

How do I backup my data? To backup your data you will need to expose port 3306 and use a database tool to dump its contents. To connect to the database the server host should be 127.0.0.1, the port is 3306, the username is acore and the password is acore.

How do I contribute? The best way to contribute is to join the community to help others and to share and discuss your creations.
