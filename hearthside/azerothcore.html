AzerothCore

Wobblin' Goblin's AzerothCore for Docker Desktop

AzerothCore for Docker Desktop is a great way to start an AzerothCore server for yourself or for your family and friends. The only requirement is Docker DesktopDocker is a platform for helping people build, share and run applications. Docker Desktop is a graphical application for building and sharing applications and services on the Docker platform. which we will install as part of the tutorial.

Minimal technical expertise is required for this tutorial outside reading some instructions and some copying and pasting. We'll mostly be relying on Docker Desktop and in-game commands to run our server so little command line knowledge is necessary.

wobgob/azerothcore is a Docker imageA Docker image is a file used to execute code in a Docker container. A Docker container is a runnable instance of an image. provided by Wobblin' Goblin with opinionated defaults to allow for a good single player or small group experience. At a later date instructions will be provided for compiling the images from source code to create custom experiences.

Prerequisites

You'll need Docker Desktop to get started. You can find installation instructions for WindowsOn Windows you'll want to use Windows Subsytem for Linux as Docker's backend. You can learn to Install Windows Subsystem for Linux on Microsoft's website., Mac, and Linux on Docker's website. This is the hardest part of the tutorial so read the installation instructions carefully and if you run into any problems don't be afraid to ask the community for help.

Installation

Once you have Docker Desktop installing the server is done in three easy steps.

  1. Pull or run wobgob/azerothcore from Docker Hub
  2. Enter the settings required to run wobgob/azerothcore
  3. Wait for the server to initialize before logging in with your 3.3.5a client

  1. Run Image

    In Docker Desktop go to the search bar (Ctrl+K) and search for wobgob/azerothcore. Once you've found the image you'll want to start with the latest tag and then hit Run which will prompt you for more input.

  2. Enter Settings

    Open Optional settings and make sure your settings match the below image then hit Run.

    In general,

    1. The container name can be anything. We've chosen to go with a randomly generated name.

    2. Host ports should match container ports.

      Warning! Exposing port 3306 can be a security vulnerability as anyone with access to that port can access your database.

    3. The host path can be anything. We've chosen to go with a volumeVolumes are the preferred mechanism for persisting data generated by and used by Docker containers. named after the container path.

      Warning! Do not delete your volume in Docker Desktop without backing it up or you'll lose all your data.

    4. The container path must be /var/lib/mysql

  3. Login

    You'll be taken to the server's logs where you'll need to wait for the server to initialize before you can login. This can take a few minutes.

    Set the realm list of your 3.3.5a client to 127.0.0.1 then use the username acore and password acore to log into the console account.

Setup

Once you're logged into the console account create a game master character with a name of your choice. This will be the character you manage the server with.

It's recommended you change the password of the console account if you're playing with others. To change the password enter the following commandSee AzerothCore's website for a list of game master commands. into chat

.account password $old_password $new_password $new_password

Note the dot (.) before the command.

It's recommended that you use a separate player account for playing by yourself or with others. To create another account enter the following command into chat

.account create $account $password

You can now log into your player account and begin playing!
